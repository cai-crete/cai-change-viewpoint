## 1. `template-layout.md` 변경 사항 파악

새로운 프로토콜 문서의 CSS 구조를 분석한 결과, 기존의 십자가 형태(3x3 Cross-Shape) 전개도 배치에서 **건축 제도 원칙에 입각한 수평 전개 T-Shape 형태(2x4 Grid)**로 레이아웃이 완전히 변경되었습니다.

**[기존 3x3 Cross 레이아웃]**
- 1행: (공백) / REAR / (공백)
- 2행: LEFT / TOP / RIGHT
- 3행: (공백) / FRONT / (공백)

**[신규 2x4 Horizontal T-Shape 레이아웃]**
- 1행: (공백) / **TOP** / (공백) / (공백)
- 2행: **LEFT / FRONT / RIGHT / REAR** (모든 입면도가 Z축(높이)을 공유하며 1열로 수평 전개)

이러한 변경으로 인해 AI가 이미지를 생성할 때의 규칙, 프런트엔드에서 5개의 입면을 잘라내는(Crop) 좌표, 그리고 캔버스 상의 자체 아트보드(CSS Grid) 코드까지 모든 파이프라인의 좌표계를 2x4 T-Shape으로 전면 개편해야 합니다.

## 2. 수행 계획 (`App.tsx` 적용)

### ① `sitePlanPrompt` 업데이트 (프롬프트 지시)
PHASE 2에서 5면 전개도를 그려주는 AI에게 제공하는 레이아웃 가이드라인 텍스트를 최신 "2x4 Horizontal T-Shape" 템플릿에 맞게 재작성합니다.
- 지시사항: "Grid Layout: 2 Rows x 4 Columns. Height of Row 2 must be identical across all faces."

### ② 오토 크롭 매핑 함수(`getElevationSlot`) 개편
UI 우측 하단의 [GENERATE]를 누를 때 선택될 도면의 그리드 좌표를 3x3에서 2x4 기준으로 수정합니다.
- FRONT: `row: 1, col: 1`
- REAR: `row: 1, col: 3`
- RIGHT: `row: 1, col: 2`
- LEFT: `row: 1, col: 0`

### ③ 단일 시트 분할 크롭(`cropCell`) 좌표 개편
AI가 생성해 준 이미지를 5개 뷰로 자르는 `generateSitePlan` 내부의 분할 로직을 수정합니다.
- 셀 크기: `cellW = img.width / 4`, `cellH = img.height / 2`
- 크롭 좌표를 T-Shape 기준 행/열 인덱스에 맞게 매핑.

### ④ 캔버스 내 5면 분리형 Grid UI 개편
원본 이미지를 클릭했을 때 우측에 나오는 아트보드(`Item-bound Library Artboard`)의 인라인 CSS `display: grid` 코드를 새 템플릿의 `grid-template-areas` 및 비율(`1fr`)과 정확히 일치하도록 수정합니다.
- `gridTemplateColumns: ${D}fr ${W}fr ${D}fr ${W}fr`
- `gridTemplateRows: ${D}fr ${H}fr`
