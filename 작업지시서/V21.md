## 1. 변경 사항 파악

### 1) Protocol V7 (`protocol-image to elevation-v7.md`) 변경 핵심
- **[ABSOLUTE SYSTEM OVERRIDE 계층 추가]:** 시각화 실행 엔진(Protocol B)에서 렌더링 시 최우선적으로 지켜야 할 **절대 제약 조건 2가지**가 강력하게 명시되었습니다.
  1. **투명 배경 강제 (Alpha Channel Mandatory):** 단 1픽셀의 단색 배경도 허락하지 않는 100% 투명 컷아웃 PNG 형식을 요구합니다.
  2. **비율 강제 동기화 (Strict Ratio Alignment):** 이관된 건축 비례 절대 파라미터(`--bldg-width`, `--bldg-depth`, `--bldg-height`)의 비율과 생성될 이미지의 캔버스 픽셀 비율이 **오차 없이 100% 일치**해야 함을 강제합니다.

### 2) Layout Template (`template-layout.md`) 변경 핵심
- **오버플로우 강제 차단 패치:** 기존 `cross-grid-container` 및 `view-panel` 내부에 물리적 크기 제한(`min-width: 0`, `min-height: 0`)을 걸어주어 내부 이미지가 비례를 깨고 그리드를 밀어내는 현상을 강력히 방어(Defense Maximized)하도록 css 로직이 패치되었습니다.

---

## 2. App.tsx 적용 계획

`App.tsx`의 렌더링 파이프라인에서 AI 프롬프트 스크립트를 V7 프로토콜의 **절대 제약 조건(Absolute System Override)**에 맞게 강력한 제어문을 포함하여 업데이트하고 패치해야 합니다.

### ① `sitePlanPrompt` 제약 조건(Prompt Override) 강화 반영
- 기존 프롬프트 하단에 V7 프로토콜의 내용을 반영하여 다음의 문구를 강제 지시문으로 주입합니다:
  - `"ABSOLUTE SYSTEM OVERRIDE 1: Alpha Channel Mandatory - The result MUST be a 100% pure transparent PNG cutout. NO solid backgrounds. Any whitebox overflow is strictly forbidden."`
  - `"ABSOLUTE SYSTEM OVERRIDE 2: Strict Ratio Alignment - The rendered building proportions must exactly, mathematically match the architectural Z-axis height, width, and depth without any arbitrary canvas scaling or distortion."`

### ② 렌더링 캔버스 오버레이 패치 (CSS 방어 로직 연계)
- 렌더링 시트가 주입되는 5면 분리 크롭 로직 및 컴포넌트(`cropElevationFromSheet`)의 `div`에 방어선 패치(`min-width: 0`, `min-height: 0`, `object-fit: cover`) 등의 스타일링이 일관되게 작동하는지 점검 및 필요시 인라인 CSS 추가. (현재 V18에서 아트보드의 CSS Grid 설정과 동일선상의 로직 정규화)
